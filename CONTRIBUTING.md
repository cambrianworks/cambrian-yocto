# Contributing to cambrian-yocto

We're excited that you're interested in contributing to cambrian-yocto. As an open-source project, we value community contributions from our partners.

## Getting Started

Before you start contributing, please take a moment to read the following:

* **Contributor Assignment Agreement:** Please review our [Cambrian Works - Contributor Assignment Agreement](<./Cambrian Works - Contributor Assignment Agreement.md>). By contributing, you agree that your submissions will be subject to the conditions of the agreement.

* **License:** Our project is licensed under **[Apache 2.0](./LICENSE.md)** ([https://www.apache.org](https://www.apache.org/licenses/LICENSE-2.0)). By contributing, you agree that your submissions will be licensed under the same terms.

## How to Contribute

We primarily accept contributions via **Pull Requests (PRs)** on GitHub.

### 1. Proposing New Features or Major Changes

For significant changes, new features, or major refactoring please contact a representative of Cambrian Works. Unsolicited pull requests may be rejected if not aligned with the future technical or product direction intended by Cambrian Works.

### 2. Fork and Clone

1.  **Fork** the repository to your own GitHub account.

2.  **Clone** your fork locally:
    ```bash
    git clone git@github.com:[your repository]/cambrian-yocto.git
    ```
    
3.  Set the original repository as the **upstream remote**:
    ```bash
    cd cambrian-yocto
    git remote add upstream github.com:[your repository]/cambrian-yocto.git
    ```

### 3. Create a Branch

Create a new, descriptive branch for your contribution. Use a clear naming convention (e.g., `feature/add-package-XXX`).

```bash
git checkout -b your-branch-name
```
